第七屆WeHelp Academy申請

個人簡介與申請動機

你好，我是白佳弘，大學主修材料工程，畢業後先在事務所擔任專利工程師，培養了精準分析與撰寫技術文件的能力。之後因想體驗不同的工作型態，也在餐飲業累積實務經驗，曾擔任吧台手、廚房助手與服務生，強化了服務意識與團隊協作能力。雖然這些經歷與軟體工程並不直接相關，但我一直對以技術解決問題抱有高度興趣。三年前我透過線上課程自學網頁基礎，主要涵蓋 HTML 與 JavaScript 的初階概念，並完成數個練習作業，從中體會到從無到有完成作品的成就感。

我希望培養一種可以帶著走的技能，不論在哪個地方或產業都能發揮價值；同時期盼未來的工作能兼具創造性，讓我透過程式設計不斷嘗試新的想法。參加這個 bootcamp 能幫助我系統化地學習軟體工程，並作為我轉職的第一步。

曾經做過哪些軟體工程技術相關的學習？若有作品請分享給我們。

大約三年前，為了轉職我開始自學前端，選擇 Udemy 的網頁全端課程，系統化學習 HTML 與 JavaScript 的基礎。當時學習偏向吸收與跟著教學操作，我完成了幾個小型練習作業（例如靜態網頁與互動式按鈕效果）作為練習。雖然現在對部分課程細節記憶有限，但仍保留一些作業檔案做為成果參考（網頁及自製的播放器，如下方連結）。隨著學習深入，我意識到自己缺乏以專案為導向的實作經驗，尚未能獨立完成一個完整專案。因此我期待透過 bootcamp 的專案訓練與同儕合作，補足實戰能力，作為轉職的關鍵一步。

如果參與這個訓練，會怎麼安排學習時間？

每週一早上先確認當週任務，下午將任務拆解成 4–5 個可行的小步驟，每個步驟的工作量以一天可完成為目標，並預留 1–2 天作為緩衝。

從週二開始，每天上午先針對當日小任務，搜尋並理解需要用到的語法與函式；下午著手實作並套用上午所學，如發現資料不足，再補充查詢。若遇到難以自行解決的問題，傍晚前會尋求 AI、同儕或導師的協助，以確保進度。

完成當日任務後，視情況選擇深入研究困難概念或提前準備隔天的任務。同時避免將整週的工作量集中在幾天內，維持穩定節奏，兼顧學習與休息，避免因過度壓縮產生報復性休息或進度延誤。

軟體技術日新月異，如何確定投入的領域是正確有回報的？

我認為沒有人能完全確定投入的領域一定會帶來回報，這和人生中多數的抉擇一樣充滿未知。但只要我持續在這個領域耕耘，所投入的心力終將成為未來追求目標的基石。唯有先穩固基礎，才有本錢在技術日新月異的環境中跟上新知。

也正因為技術持續快速演進，學習本就不可能停止。這次的密集培訓對我而言，不只是習得特定技能，更是一次建立自主學習模式與自我成長的鍛鍊，為未來持續探索與進步奠定良好的起點。

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

去年我到法國打工旅遊。剛抵達時，對自己的法文進步與生活充滿期待，認為能在當地接觸更多新奇事物，獲得靈感。然而現實並非如想像般理想：在短暫的語言學校課程結束後，密集的工作讓我語言上進步的空間有限，也逐漸發現無論國內外，人們的日常其實差別不大。我一度被繁忙的工作消磨，失去了細細欣賞我居住的歷史城市的心情。

在那段產生負面情緒的時期，我刻意提醒自己先脫離情緒的源頭或暫停相同的思考模式，避免過度陷入沮喪中。只有在稍微平靜下來之後，我才能以比較客觀的角度去檢視情緒的成因，理解自己為何失去原本的熱情。這個過程讓我能夠冷靜地分析問題，並找出可行的調整方向。

後來我重新思考此行的目的——除了體驗國外生活，也希望能到不同的地方旅行。於是我調整心態，不再把自己困在「兩點一線」的框架中，為自己規劃了幾場小旅行，走訪了幾個一直想去的城市與小鎮。同時，我也接下另一份工作，反而享受忙碌帶來的充實感。

這段經歷讓我學會接納語言學習是漸進的過程，也體會到生活的豐富與否往往取決於自己的態度與選擇。當我願意主動創造機會、調整步調時，不僅重新找回探索的樂趣，也在異國生活中培養了更強的彈性與自我調適能力。

關於這份申請網頁，分享一個開發時的技術心得。

在製作這份申請網頁的過程中，我最大的收穫是更深入理解了 RWD（響應式網頁設計） 的重要性與實作方式。原本在電腦上預覽時，版面看起來一切正常，但當我切換到手機或縮小視窗時，標題常常跑版，甚至會被壓成兩行，讓整體排版失衡。

為了解決這個問題，我開始學習使用 @media query 針對不同螢幕寬度調整樣式，例如在小於 600px 的螢幕上縮小字體、調整 padding，確保內容不會被 header 壓到。此外，也嘗試用 rem、vw 這些相對單位取代 px，讓字體與間距能隨螢幕大小動態調整，減少不同裝置上的不一致。

這次經驗讓我體會到，網頁開發不只是把內容排出來而已，更需要考慮「不同使用者實際的瀏覽情境」。未來如果繼續往這方向學習，我也會更注重跨裝置體驗，並且把「能否適應不同環境」當作檢查網頁完成度的一個必要步驟。

如何看待人工智慧的發展對軟體工程師的影響？

近來在 PTT 與 Dcard 上常看到討論 AI 發展對軟體工程師的衝擊，甚至台灣的 bootcamp 似乎也比以往減少。以一個尚未踏足資訊產業的人來看，人工智慧確實可能縮減部分傳統職缺。

這讓我想起中國棋王柯潔的經歷。2017 年，他身為世界第一的圍棋選手，卻連續敗給搭載人工智慧的 AlphaGo，從起初自信滿滿到最後無奈地說出：「人類的棋譜，其實是在汙染 AI 的資料庫。」連領域頂尖的人都必須面對 AI 帶來的變化，何況是像我這樣的入門新手。

然而，柯潔最後選擇的不是與 AI 對抗，而是與它共存。他開始主動利用 AI 復盤對局、分析自己的弱點，甚至將 AI 作為日常訓練的夥伴，藉此不斷精進棋藝。

對軟體工程師而言，AI 也不一定是競爭對手，而是可以加速學習、補足知識盲區的工具。與其害怕被取代，不如像柯潔一樣，把 AI 視為提升效率與拓展能力的助力，學會運用它，反而能在快速變動的科技環境中找到自己的價值。
